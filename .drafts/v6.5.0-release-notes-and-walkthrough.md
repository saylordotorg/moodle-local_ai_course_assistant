# SOLA v6.5.0 — Avatar animation probe (Stage 0)

Release date: 2026-06-11. A small learner-facing release: the generated SVG
avatar now blinks while idle, and the existing mouth movement plus the speaking
glow are gated behind a new admin setting with a per-course override. This is
Stage 0 of the talking-avatar evaluation in
`.drafts/sola-talking-avatars-decision-2026-06-11.md`: measure whether learners
respond to an animated coach before any paid avatar vendor is considered.

## Part 1: Changelog

**v6.5.0 — The avatar blinks, and animation becomes measurable.**

- **Idle blink.** SVG avatars (toggle button, header, picker previews) blink
  every ~5 seconds. Pure CSS, zero JS cost, only for learners who have chosen
  an SVG avatar.
- **`avatar_animation_enabled` admin setting (default on).** Gates all avatar
  animation as one unit: idle blink, the Web Audio mouth-sync during TTS, the
  CSS mouth fallback, and the speaking pulse-glow on the toggle button.
  Default on preserves the current shipped behavior.
- **Per-course override for A/B measurement.** Set the config value
  `avatar_animation_course_<courseid>` to `0` or `1` to override the site
  setting for one course. This is the Stage 0 probe mechanism: animation on in
  one course, off in a comparable one, compare TTS usage, session length, and
  repeat usage from the existing analytics.
- **Policy bundle eligible.** `avatar_animation_enabled` joins the signed
  policy bundle allowlist (now 43 keys), so the fleet can flip it without a
  deploy.
- **Reduced motion respected end to end.** CSS animations were already
  neutralized by the `prefers-reduced-motion` block; the requestAnimationFrame
  mouth-sync (which CSS cannot stop) now checks the same media query in JS.

### Backwards compatibility
No schema change. Default setting value (on) preserves current behavior
exactly; sites that never enabled TTS see only the new idle blink. The
attribute flows PHP → mustache → JS like `data-firstname`; an older cached
template (missing attribute) fails open to enabled.

### Testing
- Live three-state verification on local Moodle (Puppeteer, real module API):
  default on (blink animation computed on all rendered eyes, speaking pulse
  active), site setting off (blink `none`, pulse suppressed), per-course
  override `avatar_animation_course_2=1` beating site-off (the A/B mechanism).
- Validators 36/36; jailbreak suite 100%; PHPUnit suite green; Behat green.
- All new strings translated into 45 languages.

Plugin version `2026061012`, release `6.5.0`.

## Part 2: How an admin uses it

1. Nothing to do for the default experience: learners with an SVG avatar see
   it blink and (where TTS is used) mouth along, as before.
2. To switch all avatar animation off site-wide: Site administration → Plugins
   → Local plugins → AI Course Assistant → **Avatar animation** (uncheck).
3. To run the Stage 0 A/B: pick two comparable courses, then
   `php -r "... set_config('avatar_animation_course_101', '0', 'local_ai_course_assistant');"`
   (or `1`) per course. After 4 to 6 weeks compare TTS plays, average session
   length, and repeat sessions per learner between the arms in the analytics
   dashboard.
4. Fleet operators can flip `avatar_animation_enabled` through a signed policy
   bundle like any other allowlisted key.
