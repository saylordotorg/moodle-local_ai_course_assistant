# SOLA v5.3.26

Stub LLM provider + 25 PHPUnit tests for the four external services that call the LLM. Closes the largest remaining test-coverage gap — `generate_quiz`, `generate_flashcards`, `score_essay`, and `generate_insights` were previously untested because the test environment has no upstream API key.

## What's new

`classes/provider/stub_provider.php` — a test-only provider registered under provider id `stub` in `base_provider::instantiate()`. The stub returns canned JSON shaped for the detected prompt category (quiz, flashcards, essay, insights, generic chat). Tests flip `set_config('provider', 'stub', ...)` once, then either accept the default canned response or use `stub_provider::program_response($kind, $body)` to exercise specific parse / contract branches. `stub_provider::$throw_next = new \moodle_exception(...)` makes the next call throw, so failure-path branches are reachable.

`tests/external_llm_calls_test.php` — 25 contract tests across the four services:

- **generate_quiz** (8): happy path, clean_returnvalue round-trip, parse-error, provider-throw, count clamping (99 → 10), `<quiz>...</quiz>` extraction, malformed-row dropping, unenrolled-user rejection.
- **generate_flashcards** (5): feature-flag-disabled, no-page-content short-circuit, happy path with a real `mod_page`, provider-throw, parse-error.
- **score_essay** (6): feature-flag-disabled, too-short rejection, happy path returning criteria + revisions, provider-throw, parse-error, clean_returnvalue round-trip.
- **generate_insights** (4): admin-only rejection for plain users, no-data short-circuit (provider not called), happy path with seeded feedback, provider-throw.
- **stub provider self-test** (2): streaming chunks the canned response, fixed token-usage values.

`base_provider::instantiate()` gains a `case 'stub'` branch with a comment marking it test-only.

## What this catches

Static checks (PHPUnit before this release) didn't cover the prompt-construction → response-parsing pipeline at all. A regression that reshaped the response schema, broke the JSON parsing fallbacks, removed a feature-flag guard, or changed the round-trip contract would have shipped silently. Each of those is now blocked at CI.

## What this still doesn't catch

- LLM-side regressions where the model emits something the parser didn't anticipate (the jailbreak suite catches some of these).
- Real-browser DOM behaviour for the v5.3.5 voice flow (v5.3.27 will start on real-browser Behat).
- The `get_realtime_token` external — it calls OpenAI's REST endpoint directly rather than via the provider abstraction, so it needs its own mock and is left for a future release.

## Backwards compatibility

Drop-in upgrade from v5.3.25. New files only: `classes/provider/stub_provider.php`, `tests/external_llm_calls_test.php`. Two new lines in the factory switch (the new branch + a comment). No schema changes, no string changes, no settings changes. A production install that somehow had `provider=stub` configured would resolve to it, but every stub method is deterministic with zero upstream calls and no DB writes outside what the caller does.
