# SOLA v5.11.0 — Release Notes, Feature Summary, and Admin Walkthrough

## Part 1: Release Notes (9 June 2026)

Vendor-recommendation optimizations from the 2026-06-09 analysis. Wires the per-turn mastery classifier through a cheap structured-output model instead of the chat tier, adds Voyage AI as a first-class embedding provider plus optional Voyage rerank-2.5 as a two-stage RAG retrieval step, captures OpenAI auto-prefix cache-hit counts so dashboards see the 50% discount, and patches a production bug where Claude Opus 4.7/4.8 reject the `temperature` parameter and SOLA surfaced the empty error as "Sorry, something went wrong." No schema change; all defaults preserve prior behaviour until an admin opts in.

- **Mastery classifier routed off the chat tier.** A new **Classifier provider** setting (`mastery_classifier_provider`, default `openai`) pairs with the existing **Classifier model** (`mastery_classifier_model`, default `gpt-4o-mini`). When both are set, `conversation_classifier.php` resolves a dedicated provider via `create_for_comparison()` (API key, base URL, and temperature inherited from the matching `comparison_providers` row) instead of running the per-turn classifier on the chat-tier model. Upgrade hook 2026060900 sets `mastery_classifier_provider=openai` on existing sites; admin choice is never overwritten. Saves ~$220/month at 100k Saylor MAU vs running the classifier on Gemini 2.5 Flash.
- **Voyage AI embeddings (voyage-3.5).** New `voyage_embedding_provider` registered in the factory, with `embed_provider=voyage` as a third option alongside OpenAI and Ollama. Implements Voyage's asymmetric `input_type=query` vs `document` projection (the retriever picks the right one automatically) and the MRL `output_dimension` field (256 / 512 / 1024 / 2048). Migration is a four-config-line change plus a one-time `rag_admin.php` reindex (~$4.80 for the full 161-course Saylor corpus). `content_indexer` now reads the in-use model name from the provider via a new `base_embedding_provider::get_model()` so non-OpenAI vectors are correctly labelled in the chunk audit trail. Voyage-3.5 brings ~+4 MTEB English vs OpenAI `text-embedding-3-small`, 4x the input context (32k vs 8k), and materially better multilingual recall.
- **Two-stage RAG with Voyage rerank-2.5 (opt-in).** New `voyage_reranker` (cross-encoder, SSRF-guarded, cost-logged) wires into `rag_retriever` between cosine top-N and the prompt: top-50 cosine candidates feed a rerank-2.5 call that returns top-K by relevance. Gated by a new admin checkbox `rerank_enabled` (off by default). Settings: `rerank_apikey` (falls back to `embed_apikey`), `rerank_model` (default `rerank-2.5`), `rerank_apibaseurl`, `rerank_candidates` (default 50). Graceful fallback to single-stage cosine if the reranker is unconfigured, errors, or returns empty. Published recall lifts: +15 Recall@10 enterprise / +39% NDCG BEIR. ~$0.05/MTok billing (~40x cheaper than Cohere Rerank 3.5 for near-equivalent quality).
- **OpenAI auto-prefix cache visibility.** `openai_compatible_provider` now captures `prompt_tokens_details.cached_tokens` from the streaming usage chunk into `last_token_usage`, so per-call telemetry and cost dashboards can see the OpenAI 50% auto-prefix discount hit rate. The discount itself is automatic at billing on any prompt over ~1024 tokens with a stable prefix; this just exposes whether it is firing.
- **Claude Opus 4.7/4.8 temperature fix.** Anthropic deprecated the `temperature` parameter on reasoning-class Opus models; sending it returns HTTP 400 (`"temperature is deprecated for this model"`). The empty error body was being swallowed by the SSE write function, so the failure surfaced as the generic "Sorry, something went wrong" learner-facing string. New `claude_provider::model_supports_temperature()` per-prefix deny-list (`claude-opus-4-7`, `claude-opus-4-8`, `claude-opus-4-9`) skips the parameter for those models; Haiku, Sonnet, and older Opus are unaffected. Easy to extend as Anthropic ships more reasoning-class variants.
- **Premium escalation tier benchmark resolved (A.10).** The bake-off deferred in `sola-vendor-recommendations-2026-06-09.md` Appendix A.10 ran on 2026-06-09: 40 prompts × 4 contestants (Sonnet 4.6, Opus 4.8, Gemini 2.5 Pro, gpt-4o) × gpt-4.1 judge, 160 calls + 160 judgments, zero errors, ~$2 API spend. Verdict: Claude Opus 4.8 wins at 14.97/15 (near-perfect across every category); recommended as the premium escalation tier for the ~5% of turns matching multi-step STEM markers (hard math / CS / science). gpt-4o is disqualified (Socratic discipline collapses to 2.70/5 on hard prompts). Gemini 2.5 Pro is the price-quality Pareto winner but a 11.5-second P50 TTFT makes it unusable for real-time chat. Routing logic itself is v5.12 scope; v5.11.0 ships the bake-off evidence and the cost projection update only.
- **Benchmark harness improvements.** `admin/cli/run_tutor_golden.php` gains a `--prompts=FILE` flag (passes through to both run and judge modes) so one-off fixture sets like the A.10 prompts file can be driven without renaming the canonical `tutor_prompts.json`. `parse_comparison_providers` disambiguates labels when multiple rows share the same provider id (e.g. claude-haiku-4-5 + claude-sonnet-4-6 + claude-opus-4-8) by appending `:model-name`; `--providers=` filter does prefix match so existing `--providers=claude` invocations continue to work unchanged.

### Backwards compatibility

Default-off across the board. An install upgrades to v5.11.0 and behaves identically until an admin opts in:
- The classifier routing change is the one exception — the upgrade hook sets `mastery_classifier_provider=openai`, so any site with an OpenAI row in `comparison_providers` will start routing the classifier through `gpt-4o-mini` automatically. Sites without an OpenAI row fall back to the chat-tier model (existing behaviour); the resolve method catches the missing-key failure and degrades gracefully.
- Voyage embeddings stay off until an admin sets `embed_provider=voyage`. Switching providers requires a one-time reindex.
- Two-stage rerank stays off until an admin sets `rerank_enabled=1`.
- OpenAI cached-tokens capture is additive metric collection.
- The Claude temperature fix changes only Opus 4.7+ behaviour (which was broken before; previous callers got HTTP 400). All other Anthropic models are unchanged.

## Part 3: Admin Walkthrough (~15 min)

Audience: admin who wants to turn on the v5.11.0 optimizations on their install. Hosted Saylor production admins can do every step end-to-end without downtime; the changes are gated and additive.

### 0. Setup (1 min)

Have admin access to Site administration → Plugins → Local plugins → AI Course Assistant. Open `comparison_providers` so you can see the existing rows. Make sure you have, at minimum, an OpenAI row (for the classifier change) and a way to obtain a Voyage AI API key (for the embeddings + rerank changes; free trial at voyageai.com gets you started).

### 1. The mastery classifier change (2 min)

Site administration → Plugins → Local plugins → AI Course Assistant → scroll to the **Mastery / Objectives** section. Two settings now sit together:

- **Classifier provider** (`mastery_classifier_provider`) — new in v5.11.0, default `openai`. The id of a provider row in `comparison_providers` that handles the per-turn mastery classification call.
- **Classifier model** (`mastery_classifier_model`) — default `gpt-4o-mini`. The model name passed to that provider.

When both are set, the classifier routes through that dedicated row (API key + base URL + temperature all come from there) instead of using the chat-tier model. The default pairing is `openai` + `gpt-4o-mini` — the cheapest TIER 1 option for structured-output classification, with native JSON mode. On the existing Saylor production, the upgrade already moved the classifier off Gemini 2.5 Flash for you (it ran via the upgrade hook); saving on the order of $220 per month at 100k MAU. If you want to point the classifier at a different provider (say a cheaper self-hosted vLLM model with JSON support), change both settings, save, and the next chat turn picks up the new routing. If you leave **Classifier provider** blank, the classifier reverts to the chat-tier behaviour (the resolver does the right thing).

### 2. Voyage AI embeddings (3 min)

Get a Voyage API key (voyageai.com → trial; ~$5 free credit is plenty for a smoke). Then in Site administration → Plugins → Local plugins → AI Course Assistant → **RAG Embedding** group:

1. Set **RAG Embedding Provider** (`embed_provider`) to **Voyage AI (voyage-3.5 — recommended)**.
2. Set **Embedding API Key** (`embed_apikey`) to the Voyage key.
3. Set **Embedding Model** (`embed_model`) to `voyage-3.5` (or leave blank — provider default is the same).
4. Set **Embedding Dimensions** (`embed_dimensions`) to `1024` (voyage-3.5 default; valid MRL values are 256 / 512 / 1024 / 2048).
5. Leave **Embedding API Base URL** blank (Voyage default is correct).

Save. Now reindex: open the RAG admin page (`rag_admin.php`), pick BUS101 (or your equivalent smoke course), and click **Reindex**. ~$4.80 for the full 161-course Saylor corpus; pennies for a single course. Ask the SOLA tutor a question that needs course content; you should see citations the same way as before, but with a cleaner retrieval signal — Voyage-3.5 brings ~+4 MTEB English over OpenAI text-embedding-3-small and a much better multilingual story (relevant given ~30% of SOLA's pilot is ESL).

The chunk audit trail records the model that produced each vector, so during a cutover the chunks table will show a mix of `text-embedding-3-small` and `voyage-3.5` until the reindex completes. The retriever projects the query through whichever provider is currently configured; mixed states are safe (retrieval quality just degrades to the lower-MTEB of the two until the reindex finishes).

### 3. Two-stage RAG with Voyage rerank-2.5 (3 min)

Optional, off by default. In Site administration → Plugins → Local plugins → AI Course Assistant → **RAG Embedding** group → scroll past the embedding settings to the new **Rerank** subsection:

- **Two-stage retrieval (Voyage rerank-2.5)** (`rerank_enabled`) — check the box to enable.
- **Rerank API key** (`rerank_apikey`) — leave blank to reuse the embedding key (typical Voyage deployments share one key across embed and rerank).
- **Rerank model** (`rerank_model`) — default `rerank-2.5`.
- **Rerank API base URL** (`rerank_apibaseurl`) — leave blank to reuse the embed base URL.
- **Rerank candidate window** (`rerank_candidates`) — default 50. This is how many cosine top-N candidates feed the rerank stage. Larger windows give the reranker more material to work with at small extra cost (~10k tokens per rerank operation).

Save and ask BUS101 a question that previously gave a borderline retrieval. With rerank on, retrieval becomes two-stage: cosine selects top-50, rerank-2.5 cross-encodes (query, document) pairs and returns the top-k (per your existing **Top-K Chunks** setting), with the relevance score replacing the cosine score in the chunk telemetry. Published lifts: +15 Recall@10 on enterprise corpora, +39% NDCG on BEIR. Billing is ~$0.05/MTok — about $63/month at 100k Saylor MAU; about 6% of chat spend for a material answer-quality lift.

If anything goes wrong (network failure, malformed response), the retriever logs a debug line and falls back to single-stage cosine top-k. Nothing observable changes for the learner.

### 4. OpenAI cache-hit visibility (1 min)

No setting to flip. Any OpenAI call (chat, classifier, analytics) now writes `cached_tokens` into the per-call usage record. The OpenAI auto-prefix discount (50% off cached input tokens, fires on any prompt over ~1024 tokens with a stable prefix) is automatic at billing — v5.11.0 just makes the hit rate observable. If you want to see it now: send a few chat turns to a SOLA-enabled course, then in the database read the most recent rows of `local_ai_course_assistant_msgs` — the `cached_tokens` column should be non-zero on calls past the first one in a conversation.

### 5. Claude Opus 4.7+ temperature fix (1 min)

No setting to flip; just verification. If you have a `claude-opus-4-7` or `claude-opus-4-8` row in `comparison_providers`, prior versions of SOLA would surface "Sorry, something went wrong" when calling them — because Anthropic dropped support for the `temperature` parameter on reasoning-class Opus models and the streaming write function swallowed the empty error body. v5.11.0's `claude_provider` checks the model name against a per-prefix deny-list and omits temperature when calling those models. Test by adding an Opus 4.7 or 4.8 row to comparison_providers and pinging it from `admin/cli/run_tutor_golden.php --providers=claude:claude-opus-4-8 --limit=1` — it should return a normal response now.

### 6. Premium escalation tier (A.10) preview (2 min)

The A.10 bake-off (`.drafts/sola-a10-premium-escalation-bakeoff-2026-06-09.md`) ran on dev on 2026-06-09. Verdict: Claude Opus 4.8 is the recommended premium tier for hard math / CS / science prompts at ~5% routing rate; cost ~$700/month at 100k Saylor MAU. The routing logic itself is v5.12 scope (regex on user message + course-tag fast path), not in this release. v5.11.0 is the evidence + cost-projection update only — the headline cost table in the vendor recommendations doc now shows a baseline-plus-premium total of ~$2,070/month at 100k MAU (~30% cheaper than the current Haiku-as-primary fiction).

If you want to see the bake-off data, the artefacts are committed at `.drafts/sola-a10-bakeoff-judge-2026-06-09.csv` (160 per-prompt judge rows) and `.drafts/sola-a10-bakeoff-pareto-report-2026-06-09.md` (Pareto summary).

### 7. Benchmark harness improvements (1 min)

`admin/cli/run_tutor_golden.php` gained two niceties for one-off bake-offs:
- `--prompts=FILE` lets you drive any tutor_prompts.json-shaped file (`tests/golden/tutor_prompts_a10_premium_escalation.json` is the first non-default fixture; build more as needed).
- `parse_comparison_providers` now disambiguates labels when multiple rows share the same provider id, so you can have a `claude` row for haiku, a second for sonnet, and a third for opus — labels become `claude:claude-haiku-4-5`, `claude:claude-sonnet-4-6`, `claude:claude-opus-4-8` automatically. `--providers=claude` still matches all three (prefix match).

### 8. Q&A (rolling)

- **"Does the classifier change cost anything?"** It saves money. The previous behaviour ran the classifier through the chat tier (Gemini Flash on Saylor production); the new behaviour routes through gpt-4o-mini at ~1/40 the per-token cost for the structured-output workload. ~$220/month saving at 100k MAU.
- **"Do we have to migrate to Voyage embeddings?"** No. Default is unchanged (`openai` + `text-embedding-3-small`). Migration is a four-config-line change plus a one-time reindex. Recommended for the multilingual lift and the longer input context, but the existing OpenAI embeddings work fine.
- **"Is the reranker on by default?"** No — `rerank_enabled` defaults to 0. Opt in per site; the retriever falls back to single-stage cosine if rerank fails for any reason, so the only downside is bill creep.
- **"What's the smoke for the Opus temperature fix?"** Add a `claude-opus-4-7` or `claude-opus-4-8` row to comparison_providers, run `admin/cli/run_tutor_golden.php --providers=claude:claude-opus-4-8 --limit=1`. Before v5.11.0 you'd get an error; now you get a real response.
- **"When does the premium escalation tier ship?"** v5.12. The bake-off resolved the model choice (Opus 4.8) and the budget (~$700/mo at 100k MAU for 5% routing); the trigger logic is the next deliverable.
