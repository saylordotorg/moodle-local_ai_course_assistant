# SOLA v4.1.2 — Release Notes (CSS hotfix)

## Headline (27 April 2026)

CSS hotfix for the v4.1.1 Bootstrap 5 form-switch migration: every toggle label on `course_settings.php` was clipping its first character ("Enable RAG first" rendered as "nable RAG first") because the inline width override on `.aica-toggle` ate into the label gap. Reported by Tamás the same afternoon v4.1.1 shipped.

Plugin version `2026042704`, release `4.1.2`. Moodle 4.5+ required.

## What changed

### CSS bug fix — toggle labels no longer clipped

Bootstrap 5 lays out `.form-check.form-switch` with a fixed contract:

- wrapper `padding-left: 2.5em`
- input `width: 2em` and `margin-left: -2.5em`
- ~0.5em of breathing room before the label

The v4.1.1 inline style at `course_settings.php:754` set `.form-check-input.aica-toggle` to `width: 2.4em` for a slightly chunkier tap target. That bumps the input's right edge to `-0.1em` relative to the label — and since labels render at the wrapper's content edge with no left padding of their own, the first character lands underneath the switch pill and gets clipped.

Tamás's workaround was to delete the `<div class="form-check form-switch">` wrapper outright; that does fix the clipping but loses the pill visual and the `role="switch"` semantics.

The actual fix in v4.1.2 is to drop the `width` override entirely, keeping only the `height: 1.2em` (vertical, doesn't affect horizontal layout) and `cursor: pointer` rules. Bootstrap's default 2em width restores the 0.5em label gap. All 12 toggles on the page render with full label text.

### smartedu bypass JS — kept

The `pointerdown` handler that bypasses block_smartedu's click hijacker is no longer strictly required for any installation we still actively support, but the handler is idempotent (it does what the browser would have done) and only fires on pointer events, so keyboard accessibility (Space key) is untouched. Removing it would risk regressing whatever install made us add it in the first place; cost of keeping is ~30 lines of dead-on-most-sites JS. Decision: keep.
