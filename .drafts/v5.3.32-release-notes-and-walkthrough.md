# SOLA v5.3.32

Cron task PHPUnit coverage (batch 2 of 3). 15 new tests across 8 LLM-using / digest / anomaly / classifier tasks. The first 5 retention/cleanup tasks got coverage in v5.3.31; this batch brings the total to 13 of 17 SOLA scheduled tasks tested.

**One more `?:` fall-through bug found and fixed**, fourth in this run. `run_anomaly_digest` had `(int) (get_config('anomaly_digest_threshold_pct') ?: 50)` — setting the threshold to literal `0` ("alert on any change") silently fell through to the 50% default. An admin who deliberately set the threshold low to catch small drifts would have got the high-noise default applied. Same shape as v5.3.31's three fixes, same pattern: `($raw === false || $raw === '') ? 50 : (int)$raw`.

That covers the most commonly-changed external surface in the Learning Radar feature without a real API call.
