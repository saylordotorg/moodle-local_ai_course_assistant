# SOLA v5.1.6

Hotfix release.

## What's fixed

Learning Radar email and export deliveries (text / CSV / JSON / markdown) were rendering the literal SOLA marker tags in the body. Admins saw entries like:

> Response:
> There is currently no conversation data available for the selected period...
>
> [SOLA_NEXT]Check for data from last month||Analyze engagement trends...||Investigate student questions...||Look for common issues...[/SOLA_NEXT]

The web client (`amd/src/learning_radar.js`) parses these markers and renders the four suggestions as clickable chips below the response. The server-side delivery path (`classes/radar_delivery::format`) bypassed that client-side stripper entirely, so the raw marker leaked into every email digest, CSV export, JSON payload, and markdown report.

Fix: added a `strip_marker_tags` helper that runs over the response body before any of the four delivery formats render. Strips `[SOLA_NEXT]`, `[SOLA_SCORE]`, `[SOURCE:...]`, `[OFF_TOPIC]`, and `[NEEDS_ESCALATION]` blocks. Leaves the substantive response intact.

Verified post-fix against the exact failing payload Tom received in this morning's Learning Radar digest: marker block stripped, response body preserved, two-sentence answer renders cleanly.

## Backwards compatibility

Drop-in upgrade from v5.1.5. No setting changes, no schema changes, no behavioural changes outside the radar-export path. Single-file diff in `classes/radar_delivery.php`.

Plugin version `2026050506`, release `5.1.6`.
