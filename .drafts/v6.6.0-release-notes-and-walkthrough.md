# SOLA v6.6.0 — A/B experiment readout + prompt debug test hardening

Release date: 2026-06-12. Two admin-facing improvements: the analytics
dashboard gains a generic A/B course comparison card (the readout for the
Stage 0 avatar probe and any future per-course experiment), and the prompt
debug log format moved into a tested class shared by its writer and viewer.

## Part 1: Changelog

**v6.6.0 — Read an experiment from one page; the debug log format is now pinned by tests.**

- **A/B experiment comparison card** at the bottom of the analytics dashboard
  (`analytics.php`): pick course A and course B, get a side-by-side table of
  ten engagement metrics — enrolled learners, active SOLA users, usage rate,
  sessions, messages, avg messages per session, avg session length, returning
  users, TTS plays, TTS plays per active user — each with a B vs A delta.
  Respects the dashboard's time-range filter. Generic by design: any
  per-course experiment (avatar animation probe, prompt variant, premium
  routing trial) reads out here with no SQL session.
- **New `analytics::get_experiment_metrics()`** powering the card; TTS plays
  counted from `interaction_type IN ('openai_tts', 'xai_tts')`.
- **Fix: session statistics undercounted.** `get_session_stats()` keyed its
  query rows by userid; Moodle's `get_records_sql` silently collapses
  duplicate keys, so every message after a user's first was dropped from
  session inference. Session counts and durations on the dashboard were too
  low for any user with more than one message. Now keyed by message id.
- **Prompt debug logic extracted to `classes/prompt_debug.php`.** The entry
  formatter (previously inline in `sse.php`), the parser (previously local
  functions in `prompt_debug_view.php`), rotation, and the playground's
  simulated-chunk splitter now live in one class used by all three surfaces.
  No format change — the round trip is byte-compatible.
- **12 new PHPUnit tests** (`tests/prompt_debug_test.php`) pin the log format
  contract: write -> parse round trip (header, totals, sections, RETRIEVED
  CHUNKS with scores, history, message, attachment), newest-first ordering and
  limits, malformed-block skipping, section state resolution
  (kept/truncated/dropped/absent), rotation at 1 MiB, and sim-chunk parsing.

### Backwards compatibility
No schema change, no learner-facing change, no settings change. The debug log
format is unchanged (now enforced by tests). Existing log files parse as
before.

### Testing
- 12 new prompt_debug tests; full plugin PHPUnit suite green.
- Live Puppeteer verification of the comparison card on local Moodle: two
  seeded courses, all ten metric rows render with correct values and deltas,
  course selectors persist the selection.
- Validators 36/36; jailbreak 100%; Behat green; 17 new strings translated
  into all 45 non-English languages.

## Part 1b: Security and hardening (full code review)

A full-codebase review (security, stability, Moodle conventions, accessibility)
was run before tagging; verified critical and major findings are fixed in this
release. Highlights:

- **Analytics export (two criticals).** The Redash export no longer sends a
  wildcard `Access-Control-Allow-Origin` (CORS is now opt-in to a single
  configured origin, default off), and the feedback/survey blocks now honour
  the anonymize flag — the default export drops user agent, page URL,
  device/browser fingerprint, and raw user id, emitting only a pseudonymous
  reference. Both verified live.
- **IDOR fix:** message rating now requires the message to belong to the
  caller's own conversation.
- **XSS fixes:** the admin chat greeting renders via textContent; AI essay
  feedback is HTML-escaped; the inline i18n JSON is emitted with JSON_HEX_TAG.
- **Performance/stability:** new `(courseid, role, timecreated)` index on the
  messages table; policy-bundle application wrapped in a transaction;
  content-indexer error handling widened to `\Throwable` with a streamed
  stale-chunk cleanup; the heavy text-scan analytics queries bounded to a
  recent-message sample.
- **Conventions:** duplicate web-service definitions removed, `$plugin->supported`
  declared, `thirdpartylibs.xml` added for the bundled Chart.js.

## Part 2: How an admin uses it

1. Open the analytics dashboard (Site administration → Plugins → Local
   plugins → AI Course Assistant → Analytics).
2. Scroll to **A/B experiment comparison**, pick the experiment course and the
   control course, press Compare. The table shows both courses over the
   currently selected time range with a delta column.
3. For the avatar probe specifically: set
   `avatar_animation_course_<id>` to `0` in the control course, wait the
   experiment window, then read TTS plays per active user, session length,
   and returning users here.
