# SOLA v3.9.9 — Release Notes, Feature Summary, and 20-minute Admin Walkthrough

## Part 1: Release Notes (21 April 2026)

v3.9.9 adds a **Spend guard** (circuit breaker for LLM spend with failover) and an **LLM Optimizer** (provider recommendations by cost and student satisfaction). Also adds a "Course AI Settings" shortcut to the global settings TOC so admins can jump between site-level and course-level settings in one click.

### Spend guard

- New `classes/spend_guard.php`. Consulted by `base_provider::create_from_config()` and `voice_registry::resolve()` before every LLM call. Sub-millisecond overhead via a 60-second MUC cache on the per-period spend total.
- Caps configurable site-wide, per-course (via `course_config_manager`), and per-capability (Chat / Voice / RAG / Analytics). Period options: daily, weekly, monthly (calendar-aligned).
- **Failover chain:** one line per failover entry (`capability:label`), labels resolve against Comparison providers (for chat/analytics) or Voice providers (for voice). When the primary hits its cap, SOLA silently swaps in the next configured failover. When none is configured, requests raise a clean "budget paused" exception that SSE handlers surface as a friendly message.
- **Notifications:** site admins (or a configured recipient list) receive email at 80%, 95%, and 100% of the cap, once per threshold per period. No spam.
- **Spend status card** at the top of the Token Cost page: per-scope bars with percent consumed, colour-coded green → amber → red as thresholds are crossed.

### LLM Optimizer

- New `classes/llm_optimizer.php`. For every capability × provider × model pair with ≥ 30 messages in the trailing 30 days, compute mean $/request from the rate card, thumbs-up rate from `msg_ratings`, and a composite score (default cost weight 0.7, quality weight 0.3, both tunable in settings).
- **Monthly spend projection:** trailing 30 days at current mix, scaled to a 30-day period. Needs ≥ 7 days of data before projecting; labels confidence low / medium / high based on data density.
- **Optimizer card** below Spend status on the Token Cost page. Ranks top 3 providers per capability with cost, satisfaction, sample size, and confidence. Surfaces the currently active provider for each capability so admins can see what to change.

### Navigation polish

- Global settings TOC gets a **Course AI Settings** button next to "Back to course". Driven by the `last_courseid` user preference set on course visits. Admins can now pivot site ↔ course ↔ site without hunting for course IDs.

### Testing

- Jailbreak suite: 25/25 PASS.
- PHP lint clean on all new + modified files.
- New admin strings are English-only for this release (admin settings surface; a 45-language i18n sync is queued for a follow-up if needed).

## Part 3: 20-minute Admin Walkthrough Script

Deltas from the v3.9.8 walkthrough:

### Minute 11 (Cost visibility — replace the old Token Cost demo)

Open **Token Cost and Analytics**. Walk through four cards top to bottom:

1. **Spend status** — pick one bar, explain: "This is the chat cap. We're at 35% of $200 for this month. If it hits 80% we get an email; at 95% another; at 100% the spend guard pauses chat and falls back to our `claude-haiku` failover row."
2. **LLM Optimizer** — look at the Chat row. "Our primary is OpenAI gpt-4o. The optimizer says DeepSeek is 60% cheaper with 91% student satisfaction vs 88% for gpt-4o on 340 rated messages. Try a week of DeepSeek and come back."
3. **Cost by Category** — Voice is 40% of our bill. Decide whether to cap voice specifically.
4. **By model** — existing per-model table.

### Minute 13 (after branding)

Demo the new **Course AI Settings** shortcut at the top of the settings page. Click it to jump straight to the last-visited course's AI settings, toggle a per-course cap, save, pop back.
