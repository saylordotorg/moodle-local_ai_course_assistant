# SOLA v6.8.5: Release Notes, Feature Summary, and Admin Walkthrough

## Part 1: Release Notes (18 June 2026)

Voice-session reliability fixes, a configurable footer "explore courses" link, and a settings-page cleanup that groups related options together.

- **Voice session greets, then pauses.** The realtime voice mode now opens with one short greeting that asks how it can help and then waits, instead of launching into an unprompted answer. The microphone stays gated until the greeting finishes so startup noise or speaker echo cannot trigger a response.
- **Voice: fixed the "buffer too small" error on interrupt.** Turn-taking is now owned entirely by server-side voice activity detection. The client no longer sends a competing audio-buffer commit, which was racing the server and failing with "buffer too small ... 0.00ms" when a learner interrupted.
- **Voice: no more stray "[" in the transcript.** A leaked SOLA_NEXT control tag could flash a bare bracket into the live transcript; the transcript now holds back a partial tag until it is known not to be a control marker.
- **Footer "explore courses" link (configurable).** An optional link can appear under the Feedback link in the widget footer, driven by two settings (footer_courses_text, footer_courses_url); clearing the text hides it. The default footer label is now "Send feedback," and the Feedback link label and the feedback panel's intro line are both admin-customizable.
- **Self-hosted Whisper on/off switch.** A new "Enable self-hosted transcription" checkbox (default on) makes the self-hosted Whisper path an explicit toggle rather than URL-presence only; turning it off forces the hosted or in-browser path even when a URL is set.
- **Settings page cleanup.** The talking-avatar provider fields now sit directly under the talking-avatar pedagogy toggle they belong to; the cross-course mastery, mastery-aware starter, program-path, and learning-path defaults are grouped under Mastery tracking; and Soapbox gets its own section (after the self-hosted Whisper area) with a direct "Open rubric editor" link to its speech rubric and ESL presets.

## Part 3: Admin Walkthrough

### A. What changed this release (about 3 minutes)

1. **Footer links.** Site administration, Plugins, Local plugins, SOLA, Settings, "Footer Links" section. Set the courses-link text and URL (ships pre-filled with the saylor.org link; clear the text to hide it). Optionally override the Feedback link label and the feedback panel's intro line.
2. **Self-hosted Whisper toggle.** In the "Self-hosted transcription (Whisper)" section, the new "Enable self-hosted transcription" checkbox turns the self-hosted path on or off without clearing the URL.
3. **Soapbox section.** Just below the Whisper area, the new "Soapbox" section holds the transcription mode and an "Open rubric editor" button that lands on the speech rubric (where the General, ESL beginner, ESL intermediate, and ESL advanced presets load).
4. **Talking-avatar settings.** The provider, API key, persona, rate-card, and webhook fields now render directly beneath the "Talking Avatar" pedagogy toggle.
5. **Mastery tracking.** Cross-course mastery, mastery-aware starter, program-path, and learning-path defaults are grouped under the Mastery tracking heading.
6. **Voice.** Open a course, Voice tab, start a session: SOLA greets briefly and waits; interrupting it no longer errors.

### B. Rebranding SOLA for your institution (about 5 minutes, unchanged since v6.8.0)

1. **Open the branding settings.** Site administration, Plugins, Local plugins, SOLA, "Branding & UI" section.
2. **Set the four names.** Institution Name (full), Institution Short Name, Assistant Display Name (full product name), Assistant Short Name. Defaults are Saylor University / Saylor / Saylor Online Learning Assistant / SOLA.
3. **Save and purge caches.** The change is instant and complete: chat widget header, personalized greeting, the system prompt, every admin settings description, scheduled-task names, and the privacy/consent/email copy all switch to the new names. No code deploy, no string-file edit.
4. **Verify on a course page.** Open a course as a learner: the tab, drawer, and greeting use the new name. No leftover `[[ ]]` placeholders anywhere (the leak-guard test catches that before release).
5. **What stays.** Internal identifiers (the Moodle plugin name `local_ai_course_assistant`, the `SOLA_NEXT` chip protocol, config keys) do not change. The lowercase `saylor.org` links and emails are intentionally not rebranded.
