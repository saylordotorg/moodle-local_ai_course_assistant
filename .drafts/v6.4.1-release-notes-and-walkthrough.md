# SOLA v6.4.1 — RAG chunk-selection debug

Release date: 2026-06-11. A small, admin-only feature release that makes RAG
chunk selection visible and verifiable in the debug service. No learner-facing
change; the debug log stays off by default.

## Part 1: Changelog

**v6.4.1 — See which chunks the retriever selected.**

You can now verify, on one page, exactly which course passages the retriever
chose for a question (the "5 best pieces"), with their relevance scores — rather
than inferring it from the assembled prompt. Three additions:

- **Debug log records the selected chunks.** When `prompt_debug_enabled` is on,
  the per-turn prompt-debug log now writes a `RETRIEVED CHUNKS` block listing
  each selected chunk with its **relevance score, source (cmid), module type,
  and content**. Previously the chunks appeared only as bare `[c:N]` labels
  inside the assembled prompt, with no scores.
- **Debug viewer shows a "Retrieved chunks (RAG selection)" card.** The
  prompt-debug viewer (Site administration → Plugins → Local plugins → AI Course
  Assistant → Prompt Debug Log) renders the selected chunks as a card with a
  count badge, on the same page as the assembled prompt and the per-section
  breakdown. Because RAG-on already replaces the page dump with chunks, the
  debug view shows the chunks instead of the full page content.
- **Prompt Playground gains a live-retrieval test.** Enter a course and a test
  question and the playground runs the real retriever, showing the top-k chunks
  it selects with scores — so chunk selection can be tested and verified without
  a live chat turn.

The default retrieval count (`rag_topk = 5`) is unchanged; this release is about
visibility, not behavior. All new strings translated into 45 languages.

### Backwards compatibility
No schema change, no behavior change to the learner experience, no settings
change. The retrieved-chunks logging is gated behind the existing
`prompt_debug_enabled` admin flag (off by default), and both new admin surfaces
require `moodle/site:config`. Plugin version `2026061010`, release `6.4.1`.

### Testing

- Puppeteer: the "Retrieved chunks" card renders with score / cmid / hint, the
  summary count badge shows, and the playground live-query field loads with no
  JS errors.
- Validators 36/36; all changed PHP lint-clean; 45-language i18n complete.

## Part 2: How an admin uses it

1. Turn on **Prompt debug logging** in settings (admin only; treat the log as
   PII once enabled — it contains learner messages).
2. Have a learner (or yourself) ask a course question in a RAG-enabled course.
3. Open **Prompt Debug Log** from the plugin admin category. Expand the most
   recent entry: the **Retrieved chunks (RAG selection)** card lists the chunks
   the retriever chose, in rank order, each with its relevance score and source.
   Use it to confirm the model received the best-matching passages.
4. To test selection without waiting for a live turn, open **SOLA Prompt
   Playground**, enter the course id and a test question, and assemble — the
   live-retrieval table shows the top-k chunks the real retriever returns with
   their scores.
