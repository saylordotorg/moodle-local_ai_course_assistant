# SOLA v6.3.0 — Release Notes, Feature Summary, and Admin Walkthrough

## Part 1: Release Notes (10 June 2026)

Selfhosted speech to text: SOLA can now transcribe learner voice input on your own hardware at zero per minute cost, via any OpenAI compatible Whisper server (whisper-server Docker, speaches/faster-whisper, whisper.cpp server). Plus the full i18n catch up batch: every admin surface added between v5.11.0 and v6.3.0 now ships in all 46 locales.

- **Selfhosted Whisper STT provider.** Three new settings under "Selfhosted transcription (Whisper)": server URL, model name, optional API key. When a server URL is configured, selfhosted becomes the DEFAULT STT path; selecting a paid provider label in Active STT provider still overrides it, so hosted OpenAI/xAI STT remain fully supported. The URL accepts a bare base (`http://10.0.0.5:8000`), a `/v1` base, or a full endpoint path — SOLA normalizes to `/v1/audio/transcriptions`. Servers on private networks or plain http must be added to the existing SSRF trusted endpoints allowlist (same pattern as selfhosted LLM backends).
- **Selfhosted STT bypasses the voice spend guard** — it is free, so a voice spend cap block never pauses mic input when a selfhosted server is configured.
- **Keyless auth supported.** `transcribe.php` now sends the Authorization header only when a key is configured (most selfhosted servers are keyless behind a trusted network). The model name is passed through so faster-whisper servers can address the model they have loaded (for example `Systran/faster-whisper-small`); empty defaults to `whisper-1`, which selfhosted servers accept or ignore.
- **Usage telemetry at zero cost.** Selfhosted transcriptions log the same `[STT Transcription]` telemetry row as hosted STT; the rate card has no `selfhosted_stt` entry so the cost is correctly $0.
- **i18n catch up batch: 100 string keys into all 45 non English languages** (4,500 new translations). Covers rerank settings, the selfhosted STT settings, the emergency kill switch panel, email unsubscribe pages, prompt proportions, the prompt debug view, mastery classifier, premium escalation, the per course spend cap default, and cost anomaly settings. Every locale now passes the completeness check with zero missing keys.

### Validation work shipped alongside (docs, not plugin code)

- A.10 judge spot check: 20 responses independently rescored; 80% exact total score agreement with the gpt-4.1 judge, no ranking changes; the Opus 4.8 premium tier verdict stands (`.drafts/sola-a10-spot-check-2026-06-10.md`).

### Backwards compatibility

No schema change. With `stt_selfhosted_url` empty (the default), STT resolution is byte for byte identical to v6.2.2. New settings ship empty/off.

Note: this release was rebased over the parallel v6.2.0 to v6.2.2 line (PRs #46 to #49: RAG relevance floor and indexing robustness, raw endpoint CSP fix for full Moodle pages, and the voice STT/TTS fixes). The two voice efforts compose: v6.2.2 made real browser recordings pass upload validation, v6.3.0 lets them transcribe on selfhosted hardware.

## Part 3: Admin Walkthrough (~10 min)

> v6.3.0 adds one new admin surface (the selfhosted STT block in the Voice section). Sections 4 and 5 are carried forward from v6.1.x and unchanged.

### 0. Setup (1 min)

Upgrade to v6.3.0 and run `php admin/cli/upgrade.php`. No schema change in this release.

### 1. Stand up a Whisper server (2 min, optional but the point of this release)

On any box the Moodle server can reach (Docker):

```
docker run -d -p 8000:8000 fedirz/faster-whisper-server:latest-cpu
```

Any OpenAI compatible transcription server works — whisper-server, speaches, whisper.cpp server. CPU with the small model transcribes a 5 second clip in well under a second; no GPU needed for STT at pilot scale.

### 2. Point SOLA at it (2 min)

In SOLA settings, Voice section, "Selfhosted transcription (Whisper)":
- **Selfhosted STT server URL:** `http://<host>:8000` (SOLA appends the endpoint path).
- **Selfhosted STT model:** the model your server loaded, for example `Systran/faster-whisper-small`. Empty sends `whisper-1`.
- **API key:** leave empty unless your server requires a token.

If the server is on a private network or plain http, also add `http://<host>:8000` to **SSRF trusted endpoints** in the Security section — same allowlist used for selfhosted LLM backends. That is deliberate: an admin entered URL does not get to bypass the SSRF validator just because it is a setting.

Note the **Active STT provider** dropdown now lists "Selfhosted Whisper server (free, uses the URL below)". Leaving it blank prefers selfhosted whenever the URL is set; choosing a paid label (hosted OpenAI or xAI row) overrides selfhosted without removing it.

### 3. Mic round trip (1 min)

Open any SOLA course, click the mic, speak a sentence, release. The transcription lands in the input box. In Token Usage & Cost, the `[STT Transcription]` telemetry row appears with tokens recorded and zero cost. Pull the server's container logs if you want to see the request arrive.

### 4. Carried forward: emergency panel round trip (2 min)

As in v6.1.x: settings quicklinks, red **Emergency controls** link, disable Chat with a reason, verify the drawer pauses, restore. Audit rows record `web:<username>`.

### 5. Carried forward: analytics (1 min)

Token Usage & Cost: Cached Tokens card, By category table with RAG and Premium routing rows, per course cross links.

### 6. Q&A (rolling)

- **"Does selfhosted STT change voice conversations (Realtime)?"** No. Realtime remains OpenAI WebSocket only. Selfhosted covers the mic transcription path. A selfhosted hands free loop (STT, chat, TTS) is a future direction.
- **"What happens if my Whisper server goes down?"** transcribe.php returns a 502 and the mic shows the standard error; chat is unaffected. Point Active STT provider at a paid row for instant fallback, or fix the server.
- **"Is audio stored?"** No change: audio is forwarded for transcription and discarded; only the text enters the conversation. With selfhosted, audio never leaves your infrastructure at all.
- **"Why does the spend cap not pause selfhosted STT?"** It is free. The voice spend guard exists to stop metered spend; there is none to stop.
