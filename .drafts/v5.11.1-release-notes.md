# SOLA v5.11.1 — Release Notes (9 June 2026)

CI-only hotfix on top of v5.11.0. Cherry-picks two single-file fixes from v5.12.0 onto the v5.11.0 baseline so the `v5.11.1` tag passes the full `moodle-plugin-ci` matrix. Zero behaviour change vs v5.11.0.

- **Fix 1 (PHPDoc Checker):** `admin/cli/run_tutor_golden.php` — `mode_run()` and `mode_judge()` were missing the `@param string $promptsfile` line in their docblocks (the parameter itself was added in v5.11.0 for the `--prompts=FILE` flag used by the A.10 bake-off). PHPDoc Checker required it; failing run was 27246452269.
- **Fix 2 (PHPUnit health check):** `classes/rag_retriever.php` line 135 used the `(int) (get_config(...) ?: 50)` falsy-default pattern for the `rerank_candidates` setting. `health_check::check_no_falsy_default_get_config_pattern` flags this as an error (Moodle PHPUnit runs with `--fail-on-warning`), so the v5.11.1-with-only-the-docblock-fix re-run failed. Both fixes were already bundled into v5.12.0; this hotfix cherry-picks just the rag_retriever hunk onto the v5.11 baseline.

### Backwards compatibility

`$plugin->version` is unchanged at `2026060900` (same as v5.11.0). A site running v5.11.0 will not detect this as an upgrade (correct: there is no code-semantic change, only a 2-line docblock and a 2-line pattern refactor). The hotfix exists purely so the tagged release passes CI for downstream consumers who gate on tag-level CI status.

If you are already on v5.12.0 or v5.12.1, install v5.12.1 instead (same fixes on top of v5.12.0 content).

### Testing

- PHPDoc Checker: clean (the two errors that failed v5.11.0's CI now pass).
- PHPUnit health check: clean (the `no_falsy_default_pattern` failure that failed the v5.11.1 first attempt now passes).
- Validators 36/36, jailbreak 32/32, PHPUnit full suite unchanged from v5.11.0 except the 3 health-check failures are now green.

Plugin version `2026060900`, release `5.11.1`.
