# SOLA v6.8.32 — Soapbox presentations, WSCUC outcomes, and full i18n

## Part 1: Release Notes (11 July 2026)

First tagged release since v6.8.6. It bundles the entire Soapbox video/audio/slides build, the WSCUC-aligned outcomes reporting, document-scoped RAG, and a full 46-language translation catch-up.

### Headline
Soapbox becomes a complete **video and audio presentation activity with student-advanced slides and slide-aware feedback**; a new **WSCUC outcomes report** aggregates per-outcome achievement against a faculty-set benchmark, fed by quizzes and by rubric-scored presentations alike; RAG can ground answers on the page the learner is viewing; and all 46 languages are back to full string coverage.

### Soapbox video/audio presentations (v6.8.12 - v6.8.20)
- Per-assignment framework: `sbx_assign` / `sbx_topic` / `sbx_rec` tables; admin caps (max length default 12 min, max recordings default 3, retention default 7 days clamped to 28, video-quality preset 360p/480p/720p); server-side clamp shared by the editor and the upload validator.
- Instructor assignment editor (topics, optional PDF case, custom rubric, video or audio-only mode, informative/persuasive type, length and attempt limits).
- Presigned S3 upload pipeline (SigV4, no AWS SDK; server never handles the bytes) defaulting to the shared archive bucket under a `soapbox/` prefix.
- Browser recorder (`getUserMedia` + `MediaRecorder`, capped bitrate, min/max timer, resumable upload); student page with recorder, topic picker, and a view/download "My recordings" list.
- Off-request transcription (self-hosted Whisper preferred) + rubric scoring; retention + stored-attempts cleanup task; course-nav links and copyable per-assignment student URLs.
- Privacy provider covers `sbx_rec` incl. the transcript (metadata, export, erasure with storage-object deletion).

### Soapbox slides + slide vision (Phase 2, v6.8.21 - v6.8.31)
- Slides data model + timeline normalizer; PDF deck upload and finalize; in-recording slide viewer with advance capture; server-side deck rasterization via Ghostscript; student-page slides wiring; synced slide playback beside the recording.
- Slide-aware scoring: per-slide text and the advance timeline feed the rubric so feedback addresses slide design, slide-to-speech alignment, and pacing.
- Optional slide-vision feedback (v6.8.31): a single gpt-4o-mini vision pass over the rendered slide images adds one short visual-design note. Off by default and dual-gated (site toggle + per-assignment opt-in); no slide images are stored.

### WSCUC outcomes and mastery (v6.8.28 - v6.8.30)
- Per-outcome achievement report: faculty-set benchmark per outcome (default 70%), reporting the percentage of learners meeting it, with an accreditation export. Generic across quizzes and Soapbox.
- Partial-credit (continuous) mastery observations: a 0-1 score contributes fractionally to the Bayesian mastery estimate instead of a binary right/wrong.
- Rubric criterion → course outcome mapping: a scored presentation records a partial-credit attempt against the mapped outcome, so it feeds the same outcomes report as quizzes.

### RAG, privacy, and embedding fixes (v6.8.7 - v6.8.11)
- Document-scoped retrieval: answers ground on the document the learner is viewing, falling back to course-wide only when the page has nothing relevant (`rag_scope` setting); reindex screen warns about documents that index to nothing.
- Embedding width defaults to provider-native (unblocks Voyage); Voyage MRL width guarded.
- Article 15 export: mastery attempts and struggle signals now included in a subject-access export.

### i18n and compliance (v6.8.32)
- Translation catch-up: 115 keys added since the v6.3.0 sync (outcomes, Soapbox video/slides/vision, RAG scope, privacy metadata, analytics-JS labels, vendor-DPA table) translated into all 45 non-English languages. 46/46 locales complete; placeholders and branding tokens preserved (validated zero drift); branding leak-guard and completeness test green.
- Soapbox DPIA: `.drafts/sola-soapbox-dpia-2026-07-11.md` documents the video/audio/slides processing, recipients (incl. the transient slide-vision pass), lawful basis, minimization, retention, and SAR/erasure coverage. Signed off by the CDO as acting data-protection authority; residual risk low.

### Testing
- CI green on every merged PR (#114-#143): PHP 8.1/8.2/8.3 x mariadb/pgsql on MOODLE_405_STABLE, plus Claude review.
- Static security/jailbreak corpus (validator suite) 36 of 36 pass. Note: the live jailbreak run (against a real provider) was not executed for this tag because the local dev Moodle has no provider key configured; the jailbreak corpus and the English system prompt are unchanged since v6.8.6, and the v6.8.7-v6.8.32 delta does not touch prompt-injection surfaces.
- PHP lint clean on every modified file; all 45 lang files lint clean.
- Deploy + BUS101 smoke on the dev fleet as the final step.
