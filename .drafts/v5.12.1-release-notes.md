# SOLA v5.12.1 — Release Notes (9 June 2026)

CI-only hotfix on top of v5.12.0. Cherry-picks the docblock fix that landed on main as commit a80644b, so the `v5.12.1` tag passes `moodle-plugin-ci phpdoc --max-warnings 0`. Zero behaviour change vs v5.12.0.

- **Fix:** `admin/cli/run_tutor_golden.php` — `mode_run()` and `mode_judge()` were missing the `@param string $promptsfile` line in their docblocks (the parameter itself was added in v5.11.0 for the `--prompts=FILE` flag used by the A.10 bake-off; the same docblock omission was carried into v5.12.0). PHPDoc Checker required it; failing run was 27245200343.

### Backwards compatibility

`$plugin->version` is unchanged at `2026060901` (same as v5.12.0). A site running v5.12.0 will not detect this as an upgrade (correct: there is no code-semantic change, only a 2-line docblock). When deployed via `deploy_dev.py`, the files are rsync'd in place; no Moodle upgrade hook runs because the version int is unchanged.

This is the version dev should run after the deploy: same v5.12.0 features (premium escalation router, etc.) with the docblock fix included.
