# SOLA v4.12.0 — Release Notes, Feature Summary, and 20-minute Admin Walkthrough

## Part 1: Release Notes (30 April 2026)

Structural prompt refactor that completes the simplification arc started in v4.11.0. Closes Tomi Molnár's "context homogeneity" critique by replacing linear concat-in-place with category-grouped `PromptSection` objects assembled within an admin-configurable token budget.

- New `classes/prompt/section.php` and `classes/prompt/builder.php` — sections carry name, category (identity / context / learner / behavior / markers / safety), priority, content, and optional truncation floor. The builder emits sections in canonical category order; under budget pressure it drops or truncates lowest-priority candidates first while always preserving the safety category in full.
- `context_builder::build_system_prompt` rewritten to declare ~14 named sections then delegate. Behaviour is preserved on the happy path; budget pressure now manifests as deterministic section drops rather than silent overflow.
- New `prompt_budget_chars` admin setting (default 8,000 chars / ~2,000 tokens). Lower values trade detail for per-turn cost.
- New `prompt_verbosity` admin setting (concise / standard / verbose, default concise) with per-course override `prompt_verbosity_course_<id>`. Concise = what hosted models follow reliably; verbose = explicit scaffolding for weaker self-hosted models.
- Socratic-mode and external-resources blocks lifted into helpers that respect verbosity.
- `sse.php` prompt-debug log now renders a category-grouped breakdown showing dropped/truncated sections.
- 7 new lang strings; 45 placeholder locales synced.

**Measured impact (TEST101 dev course)**: 16,336 chars (v4.11.0) → 7,908 chars (v4.12.0 default budget). 51% per-turn reduction. At realistic 30% adoption full bundle on Together AI Llama 3.1 8B, ~$875/month → ~$430/month.

### Testing

- PHP lint clean on every modified file.
- Validator suite: 30/30 pass.
- Empirical prompt size measurement: confirmed budget enforcement at 4,000-char setting truncates `base_template` and drops 9 lower-priority sections while keeping security in full.
- i18n: 45 placeholder locales synced.
- a11y: every new admin setting flows through Moodle's `admin_setting_*` wrappers (auto-generated label + aria-described-by). No custom UI introduced.
- Deployed and verified on dev.sylr.org, dev405.sylr.org, dev500.sylr.org (Moodle 5.0), dev501.sylr.org (Moodle 5.1), dev503.sylr.org.

Plugin version `2026043050`, release `4.12.0`.

## Part 3: 20-minute Admin Walkthrough

This walkthrough is the demo script for showing v4.12.0 to instructors, ops leadership, or external collaborators. It covers the structural prompt refactor end to end.

### 0. Setup (1 min)

Sign into a course as an admin. Open SOLA via the side tab. Confirm the widget loads and a chat is possible.

### 1. The "before" picture (3 min)

Frame the problem to the audience: every chat turn we send the model a system prompt plus the user message plus the conversation history. Pre-v4.11, that system prompt was assembled by linear concatenation of ~21 distinct instruction blocks. Tomi Molnár's audit measured it at ~3,500 tokens before the user even typed anything — and the blocks were heterogeneous: identity mixed with course content mixed with style guidance mixed with markers mixed with safety. Models read that as one undifferentiated stream and weight everything equally.

Show the v4.11 changelog entry on the wiki to land the framing.

### 2. Turn on the prompt-debug log (2 min)

Site administration → Plugins → Local plugins → AI Course Assistant → main settings page. Tick **"Log assembled system prompt to file"** (`prompt_debug_enabled`). Save.

Send one chat turn from a student account in the demo course. SSH to the server (or open the demo file viewer if available) and `tail` `moodledata/temp/sola_prompt_debug.log`. Read out the new format:

```
=== 2026-04-30 12:00:00 courseid=2 userid=2 provider=openai ===
Total: 7908 chars (~1977 tokens)
Budget: 8000 chars
Sections (by category):
[identity] 3181 chars total
     3181  base_template
[context] 100 chars total
      100  faq
[learner] 1817 chars total
     1817  personalization
[behavior] 1463 chars total
      320  role
      543  house_style
      300  multilingual
      300  practice_scoring
[markers] 599 chars total
      599  output_markers
[safety] 1443 chars total
     1443  security
```

Point out: every section is labelled, every category has a total, and you can see exactly where the bytes go. This is the measurement that was missing pre-v4.11.

### 3. The structural change (5 min)

Walk through the categories in order:
- **Identity** — who SOLA is, what course this is for. Always stays.
- **Context** — course content (RAG chunks or full stuffing), FAQ, current-page hint.
- **Learner** — first-name personalisation, learning profile, mastery state, study plan.
- **Behavior** — role, house style, Socratic mode, scoring, multilingual handling, widget feature awareness, external resources.
- **Markers** — single consolidated block listing `[[c:N]]`, `[SOURCE:...]`, `[SOLA_NEXT]`, `[OFF_TOPIC]`, `[NEEDS_ESCALATION]`.
- **Safety** — security and anti-injection. Always stays in full; never truncated.

Each section has a priority. The builder emits in category order; within a category by descending priority. When the assembled prompt would exceed the configured budget, the builder drops or truncates the lowest-priority candidates first. Safety is exempt.

### 4. Live budget demonstration (4 min)

Settings → **System prompt character budget** (`prompt_budget_chars`). Default is 8,000. Change to 4,000 and save.

Send another chat turn from the student account. Tail the log:
```
Total: 4036 chars
Budget: 4000 chars
Sections (by category):
[identity] 2592 chars total
     2592  base_template [TRUNCATED]
[context] 0 chars total
        0  faq [DROPPED]
...
[safety] 1443 chars total
     1443  security
```

Point out:
- Safety preserved in full at 1,443 chars.
- Base template truncated, not dropped — high priority means it stays but gets shortened.
- 9 lower-priority sections dropped.
- The behaviour is deterministic, not silent.

Restore the budget to 8,000.

### 5. Per-course verbosity (3 min)

Default verbosity is **concise** — the 1-line Socratic directive that hosted models follow reliably. Demonstrate this by inspecting the `socratic_mode` section in the debug log on a Socratic course; it's ~330 chars.

For a course on a weaker self-hosted model (Llama 3.1 8B running locally on Ollama, for instance), you might want explicit do/don't scaffolding. Set `prompt_verbosity_course_<courseid>` to `verbose` via the CLI config tool or per-course settings page. Reload, send a chat, and observe the same section now ~2,200 chars. The model sees explicit examples of correct and incorrect Socratic behaviour.

The opposite case — concise — is the default for production hosted models because they handle the directive without scaffolding and the saved tokens compound across millions of turns.

### 6. Cost framing (2 min)

Pull up the Token Usage analytics dashboard. Show the rolling per-day token spend curve. Note the inflection point on the deploy date: a clear step-down from the v4.11 baseline. At 30% realistic adoption full bundle on Together AI Llama 3.1 8B, the projected monthly cost dropped from ~$875 to ~$430.

If the audience is finance/ops: that's a $5,400 annual saving from the structural refactor alone, on top of the v4.11.0 consolidation savings.

### 7. The deferred work (1 min)

Explicitly call out what's NOT done in v4.12.0:
- A per-section graphical analytics surface (per-day average tokens by category) — deferred to v4.13.0 or later if the debug log proves insufficient.
- Provider-side prompt caching (Anthropic + OpenAI both support this) — separate optimisation, doesn't depend on the structural work.
- Auto-tuning of `prompt_budget_chars` from observed traffic — manual today.

### 8. Q&A (rolling)

Common questions and tight answers:

- **"Will this break courses on weaker models?"** Default verbosity stays at concise; flip per-course to verbose if a specific course needs the scaffolding. Toggle is live without restart.
- **"What if a course's content is bigger than the budget?"** Course content has a 400-char minimum (won't drop) and high priority (won't drop). It can be truncated. Use RAG to land the relevant 1-2 chunks instead of stuffing all course content.
- **"How do we know the right budget?"** Turn on prompt_debug_enabled for one week, look at the log, see what sections actually run, raise/lower the budget accordingly. The default 8,000 was tuned on TEST101; production courses with bigger FAQs or learner profiles may want 10,000-12,000.
- **"Does this affect the learner experience?"** Not directly — the learner-facing behaviour is unchanged. The model still answers, asks Socratic questions, escalates support tickets, and emits SOLA_NEXT chips the same way. The change is upstream of the model.
