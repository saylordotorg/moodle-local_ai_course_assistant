# SOLA v6.0.1 — Release Notes (10 June 2026)

Security + hardening hotfix from a five-agent parallel review (code quality, security, performance, UI/UX, test coverage) of everything shipped in v5.11.0 through v6.0.0. One real security fix, one correctness fix, two performance fixes, 14 new regression tests. No schema change beyond one new index; no settings change; no behaviour change visible to learners except the leak fix.

- **SECURITY FIX: premium-router telemetry leaked into learner chat history and the LLM context window.** `conversation_manager::get_messages()` returned every row for a conversation, including the `role='system'` telemetry rows that the v5.12 premium router writes (`[PremiumRouter] trigger:<regex>`). Those rows leaked into (a) the learner-facing history web service — the drawer would render the raw telemetry string — and (b) `get_history_for_api()`, which builds the LLM context, so the model's prompt included SOLA's routing internals. Fixed with a `role IN ('user','assistant')` filter; telemetry rows stay in the DB for analytics but never reach learners or the model.
- **Fix: spend-alert test CLI bypassed the unsubscribe contract.** `admin/cli/send_spend_alert_test_email.php` (new in v6.0.0) sent test emails without consulting `email_optout::is_opted_out(TYPE_SPEND_ALERT)`, unlike the production paths in `spend_guard` and `cost_anomaly_detector`. The CLI now skips opted-out recipients and reports them separately.
- **Performance: composite `(role, timecreated)` index on the msgs table.** The v6.0.0 `cost_anomaly_check` daily scan queries assistant messages in a 24-hour window; without this index that query degrades to a full table scan that grows with corpus size. Added to `install.xml` plus an idempotent upgrade hook (`2026061000`, guarded by `index_exists`).
- **Performance: N+1 fix in the anomaly email's per-course breakdown.** `cost_anomaly_detector::per_course_spend_for_day()` looked up each course shortname with its own query; now one `get_records_list` call for all of them.

### New regression tests (14)

- `tests/conversation_manager_filter_system_test.php` (4 tests): system rows filtered from both `get_messages` and `get_history_for_api`; `[PremiumRouter]` text never reaches learner or LLM; chronological ordering preserved with the filter.
- `tests/claude_provider_temperature_denylist_test.php` (10 tests): locks the v5.11.0 deny-list contract — `claude-opus-4-7/4-8/4-9` (including date-suffixed aliases) reject temperature; older Opus, Sonnet, Haiku, and legacy claude-3 models still accept it; empty model name defaults to accepting.
