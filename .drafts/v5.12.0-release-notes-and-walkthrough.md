# SOLA v5.12.0 — Release Notes, Feature Summary, and Admin Walkthrough

## Part 1: Release Notes (9 June 2026)

Premium escalation tier routing (the v5.11.0 A.10 bake-off follow-on). New `premium_router` class evaluates each chat turn against an admin-configured regex trigger list (default ships with multi-step STEM markers) plus a course allowlist; matching turns route to Claude Opus 4.8 by default instead of the workhorse chat tier. Off by default. No schema change.

- **Premium escalation router (`classes/premium_router.php`).** Per-turn routing decision with two trigger paths: (1) regex match on the user message against an admin list (default ships with the curated set from the A.10 bake-off — "derive", "prove that", "step by step", LaTeX math `$...$`, fenced code blocks, big-O, integrals, optimization, thermodynamics, free-body diagram, etc.), and (2) course-allowlist prefix match against course shortname or idnumber (use for STEM-heavy courses where every turn auto-escalates). Course-tag path takes precedence; regex evaluated only when course-tag misses. Falls back to the workhorse provider on any router or instantiation failure.
- **Six new admin settings under "Premium escalation tier (A.10)" heading.** `premium_escalation_enabled` (default off), `premium_escalation_provider` (default `claude`), `premium_escalation_model` (default `claude-opus-4-8`), `premium_escalation_triggers` (textarea of regexes, blank = default set), `premium_escalation_course_tags` (textarea of course-shortname/idnumber prefixes), all wired into `sse.php` between standard provider construction and the chat-completion call. Admin-LLM-picker overrides always win.
- **Per-decision telemetry.** Every escalation decision writes a `[PremiumRouter]` row into `local_ai_course_assistant_msgs` with the trigger reason (`course_tag` or `trigger:<regex>`) and the resolved model. Visible in Learning Radar / Redash so admins can monitor the actual escalation rate per course and refine triggers without guessing.
- **PHPUnit coverage (16 tests, 36 assertions).** Trigger-regex matcher (defaults match expected STEM markers; defaults do not match neutral messages; admin override replaces defaults; blank override falls back to defaults; comment lines ignored; malformed regex does not throw), course-allowlist resolver (shortname prefix, idnumber prefix, miss case, blank case), end-to-end `decide()` (returns resolved provider when trigger matches, course-tag takes precedence over message match, no-match does not escalate, empty message does not escalate, disabled by default).

### Backwards compatibility

Premium routing is opt-in via `premium_escalation_enabled`. An existing site upgrades to v5.12.0 and behaves identically until an admin checks the box. The routing logic is gated entirely behind the enabled flag so the cost of evaluating triggers on every turn is paid only when admins want it.

Cost at expected 5% escalation rate (~$700/month at 100k Saylor MAU on Opus 4.8) is on top of the existing baseline chat spend. The consolidated cost table in `.drafts/sola-vendor-recommendations-2026-06-09.md` already reflects this (~$2,070/mo total at 100k MAU with premium tier enabled).

### Testing

## Part 3: Admin Walkthrough (~10 min, premium escalation focus)

Audience: admin who wants to turn on premium escalation routing for a course (or every course). Hosted Saylor production admins can do the steps end-to-end without downtime; the change is gated behind the new enabled flag.

### 0. Setup (1 min)

Have admin access to Site administration → Plugins → Local plugins → AI Course Assistant. Confirm `comparison_providers` has a row whose first column is `claude` (it should — Claude Haiku 4.5 is in the standard SOLA stack). The Claude API key on that row is the one the premium router will reuse for Opus 4.8 calls. Have a smoke course handy (BUS101 on dev) where you can paste a few prompts.

### 1. Why this exists (2 min)

The 2026-06-09 A.10 bake-off (`.drafts/sola-a10-premium-escalation-bakeoff-2026-06-09.md`) compared four premium-tier candidates on 40 hard prompts (multi-step calc, CS proofs, deep philosophy, multi-paragraph science). Claude Opus 4.8 won at 14.97/15, with the biggest gap to gpt-4o (12.68/15) on hard math (3.0 point gap), hard CS (3.0), and hard science (3.2). gpt-4o's Socratic discipline collapsed to 2.70/5 on these prompts — it gives the answer instead of coaching. Gemini 2.5 Pro is the price-quality Pareto winner but its 11.5-second first-byte latency disqualifies it for real-time chat.

So the recommendation: route only the ~5% of turns that match multi-step STEM markers (or every turn in known STEM courses) to Opus 4.8. The rest stays on Gemini 2.5 Flash (the workhorse, 14.35/15 on the broader 06-04 benchmark) at 1/23 the per-call cost. Expected blended cost lift at 5% rate: ~$700/month at 100k Saylor MAU.

### 2. Enable the router (2 min)

Site administration → Plugins → Local plugins → AI Course Assistant. Scroll to **Premium escalation tier (A.10)** heading. Settings:

- **Enable premium escalation routing** — check it.
- **Premium provider** — `claude` (default; matches the row id in `comparison_providers`).
- **Premium model** — `claude-opus-4-8` (default; per the A.10 verdict).
- **Premium trigger regexes** — leave blank to use the curated default set from the A.10 bake-off. Or paste one PCRE regex per line (without delimiters; case-insensitive matching is applied automatically). Default set includes `\bderive\b`, `\bprove\s+that\b`, `\bstep[\s-]*by[\s-]*step\b`, `\$[^\$\n]{2,}?\$` (inline LaTeX), triple-backtick fenced code blocks, `\b(integrate|integral|derivative|differentiate)\b`, optimization, stoichiometry, thermodynamics, big-O, complexity proofs, and the LaTeX macro family `\int`, `\sum`, `\frac`, `\sqrt`, `\partial`, `\nabla`, `\infty`.
- **Premium course allowlist** — leave blank to escalate based on regex only. Or paste one course shortname/idnumber prefix per line (use this for STEM-heavy courses where every turn should escalate). Matching is case-insensitive prefix: `MATH` matches MATH121, MATH205, MATH301, etc.

Save. The router is now live.

### 3. Smoke it on BUS101 (2 min)

Open BUS101 (or your equivalent smoke course) and the SOLA drawer. Send:

- "What's the deadline for the next assignment?" — expect workhorse (no STEM markers, BUS101 not in allowlist).
- "Derive the formula for compound interest step by step." — expect premium (matches `\bderive\b` AND `\bstep[\s-]*by[\s-]*step\b`).
- "I see `$E = mc^2$` in the notes — what does it mean?" — expect premium (LaTeX inline math).
- "What's the big-O complexity of this sort?" — expect premium (big-O regex).

To confirm which turns escalated, query `local_ai_course_assistant_msgs` for `interaction_type = 'premium_route'` rows in the last 5 minutes. Each shows the conversation id, courseid, and the reason (`trigger:<regex>` or `course_tag`).

### 4. Add a STEM course to the allowlist (1 min)

Find a course shortname/idnumber to allowlist (say MATH121). Open the **Premium course allowlist** setting and add a new line: `MATH121` (exact match) or just `MATH` (prefix match for MATH-prefixed courses). Save. Now every chat turn in MATH121 escalates to Opus 4.8 regardless of regex. The reason logged is `course_tag` instead of `trigger:<regex>`.

### 5. Monitor and tune (2 min)

In the database (or via Redash if you have the Learning Radar dashboards):

```sql
SELECT
  c.shortname,
  COUNT(*) AS escalations,
  SUM(CASE WHEN m.message LIKE '[PremiumRouter] course_tag' THEN 1 ELSE 0 END) AS by_course_tag,
  SUM(CASE WHEN m.message LIKE '[PremiumRouter] trigger:%' THEN 1 ELSE 0 END) AS by_regex
FROM mdl_local_ai_course_assistant_msgs m
JOIN mdl_course c ON c.id = m.courseid
WHERE m.interaction_type = 'premium_route'
  AND m.timecreated > UNIX_TIMESTAMP() - 86400
GROUP BY c.shortname
ORDER BY escalations DESC;
```

Use this to:
- Confirm the escalation rate stays in the ~5% target band per course (divide escalations by total chat turns from the same course over the same window).
- Spot regex triggers firing too aggressively (a non-STEM course showing thousands of escalations - tighten the regex list).
- Spot allowlist gaps (a course where 30%+ of turns match the regex - strong candidate for the allowlist instead).

### 6. Roll back (1 min)

To turn off cleanly: uncheck **Enable premium escalation routing** and save. The router returns `{escalate: false, reason: 'disabled'}` immediately; no Opus 4.8 calls fire. The trigger list and course allowlist stay in their settings (no work lost), ready for the next time you flip the switch.

### 7. Q&A (rolling)

- **"How much will this actually cost?"** Depends on the escalation rate. At 5% of chat turns at 100k Saylor MAU, expect ~$700/month on top of the ~$1,370/mo baseline chat spend. At 10% (STEM-heavy course set), ~$1,400/mo. Tune the regex list to keep the rate where you want it.
- **"Can I use a different premium model?"** Yes — change **Premium model** to anything Anthropic accepts. The bake-off resolved Opus 4.8 as the quality winner, but Claude Sonnet 4.6 is cheaper if you want a less expensive premium tier (14.40/15 on the A.10 bake-off vs Opus 14.97). Edit the `premium_escalation_model` setting; no code change needed.
- **"Why not gpt-4o?"** The A.10 bake-off ranked gpt-4o LAST on hard prompts (12.68/15). Its Socratic discipline collapses to 2.70/5 — it gives the answer rather than coaching. Despite being a tier above gpt-4o-mini, it is an answer-first model, wrong shape for a tutoring escalation.
- **"Why not Gemini 2.5 Pro?"** Pareto-equal to Opus on quality and much cheaper, but the 11.5s P50 first-byte latency breaks the streaming chat UX. Pro is fine for async batch work but not for real-time chat. Use Opus for chat escalation; use Pro for the weekly benchmark harness or background analysis.
- **"What if the premium row fails?"** The router catches the failure and falls back to the workhorse provider. The learner sees a normal chat response on the workhorse model. A debugging log line records the failure for the admin.
- **"Does the admin-LLM-picker override this?"** Yes. If an admin uses the side-by-side LLM picker on a course page, explicit admin choice wins; the router is skipped entirely on that turn. Premium routing only fires on the standard student chat path.
