# SOLA v5.1.0

Minor release adding one admin setting in response to feedback from Tomi Molnár about prompt size on cost-sensitive deployments. Code change is small and isolated; default behaviour is unchanged. v5.0.0 (May the Fourth Edition) remains available as the prior release.

## What's new

**`current_page_content_maxchars` admin setting.** Caps the size of the `## Current Page Content` section independently of the overall `prompt_budget_chars` budget.

- Default 12,000 characters (matches the prior v5.0.0 hardcoded behaviour).
- Clamped to the range 500-12,000 in code so a misconfigured 0 or negative value cannot silently suppress the section.
- Cost-conscious admins running paid hosted providers can clamp lower (e.g. 3,000-4,000) to reduce per-turn token spend, at the small risk that a question whose answer is in the truncated tail of a long page will be answered less accurately.
- Independent of `prompt_budget_chars`: this caps just the page section; the budget caps the whole prompt.
- Available at Site administration → Plugins → AI Course Assistant → Settings, just below the existing `prompt_budget_chars` setting.

## Why a setting rather than smarter retrieval

The original feedback was to use RAG-only retrieval instead of injecting the full page. Our v5.0.0 UT cycles showed full-page injection has materially better page-grounded accuracy than retrieval-only on short specific questions: a chunk index can miss the relevant passage in ways the structured page injection never does. RAG remains available as an *additive* layer (course-content retrieval, when the embedding provider is configured). This setting lets each institution make its own cost/accuracy trade-off rather than forcing one architectural choice on every deployment.

## Backwards compatibility

Zero breaking changes. Every existing v5.0.0 install continues working with no admin action. The new setting only kicks in if an admin opts to lower it.
