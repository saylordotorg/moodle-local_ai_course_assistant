# SOLA v6.6.2 — admin discoverability + clearer starters, two stability fixes

Release date: 2026-06-12. A small UX + maintenance release. No schema change,
no learner-data change; safe one-jump upgrade from any v6.x.

## Part 1: Changelog

**v6.6.2 — Make the admin pages and the "back to starters" action easy to find.**

- **Course "More" menu links (v6.6.1).** Admin feedback was that SOLA's settings
  and analytics pages were hard to find, buried under Site administration. The
  course navigation ("More" menu) now shows **Course AI Settings** (staff with
  `local/ai_course_assistant:manage`) and **AI Tutor Analytics** (staff with
  `:viewanalytics`), linking to the per-course pages. Gated by the exact
  capabilities the target pages enforce, so learners see nothing. Reuses
  existing strings — no new translations. New
  `local_ai_course_assistant_extend_navigation_course` callback in `lib.php`.
- **Home button relabelled "Conversation starters" (v6.6.2).** The header home
  icon brings the original conversation-starter chips back without clearing the
  chat, but its tooltip said "Start over" — which read like a wipe and didn't
  signal what it does. It now uses `chat:starters_label` ("Conversation
  starters"), already translated in all 45 languages, so zero new i18n work.
- **Docs:** README and wiki gain a "Finding the admin pages" section (the hub
  bookmark `admin/category.php?category=local_ai_course_assistant`, admin
  search, and the course More menu). Note recorded that Moodle cannot show a
  Settings link for local plugins on admin/plugins.php (core hardcodes it to
  null for the local type), so that route is not available.

### Stability fixes (rode in with the above)

- **Prompt-debug log now rotates correctly.** `prompt_debug::write_entry`
  trusted PHP's per-process stat cache, so once the debug log grew past the
  rotation threshold within a request the size check could read stale and never
  truncate — the log could grow unbounded. Added `clearstatcache()` before the
  check. (Also fixed a deterministic CI failure in `prompt_debug_test`.)
- **Analytics experiment form** had an empty `action` attribute in the template
  example context (invalid HTML, failed Mustache Lint); the example now defines
  `experiment_form_action` (runtime already passed it).

### Backwards compatibility
No schema change, no settings change, no learner-facing behavior change beyond a
button tooltip. `chat:reset` ("Start over") is retained for compatibility.
Plugin version `2026061017`, release `6.6.2`.

### Testing (release gate on the tag)
- Full PHPUnit plugin suite green.
- Security validator corpus 36/36.
- i18n completeness: 46/46 locales, zero missing keys (no new strings).
- Jailbreak suite unaffected — no system-prompt or security code changed in
  this release (diff is course nav, a button tooltip, log rotation, an
  analytics template fix, and the rebuilt chat AMD bundle).
- Live Puppeteer checks: course More-menu links present with correct URLs and
  both target pages render; home button title/aria read "Conversation starters".

## Part 2: How an admin finds the pages now

1. Bookmark the hub: `https://<your-site>/admin/category.php?category=local_ai_course_assistant`.
2. Or type "AI Course" in the Site administration search box.
3. On any course page, open the course **More** menu — staff see **Course AI
   Settings** and **AI Tutor Analytics** there.
